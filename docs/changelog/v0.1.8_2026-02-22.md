# 版本更新记录 v0.1.8

**更新日期**: 2026-02-22

## 新增功能与架构调整 (Features & Architecture)

- **单点登录（SSO）流程打通与跨域支持**:
  - 更新 NextAuth 配置，在 `auth.ts` 中新增 `redirect` 回调，允许从外部业务站（例如 `localhost:3000`）发起的登录进行跨域跳转。
  - 修改 SSO 登录展示页 `app/page.tsx` 与登录表单 `login-form.tsx` 组件逻辑。支持获取 URL 中的 `callbackUrl` 参数，在用户已具备登录态及成功认证后优先返回发源地址，替代原本默认跳转至单一 Dashboard 的逻辑。

- **后端用户数据脱敏去中心化**:
  - `apps/backend/app/models/user.py`：精简 `UserInfo` 模型。移除本地所有的隐私信息（个人邮箱、租户、昵称等字段），只保留业务处理必须的核心 `user_id` 和鉴权 `role`。
  - `apps/backend/app/core/auth.py`：清理 `SSOAuthProvider` 解析参数环节，不再捕获解析 `username` 与 `email`，让系统明确依赖独立 SSO 保管敏感信息。

- **前端展示与安全层解绑**:
  - `apps/web/src/pages/UserProfilePage/index.tsx`：清除所有个人信息表单本地保存接口与交互，将修改操作限制在展示模式（设置为只读），在页面剥离了相关的更改与储存状态。
  - `apps/web/src/pages/HomePage/Header.tsx` 及 `Hero.tsx`：将系统所有的“登录”和“开始使用”入口统一改为跳转第三方 Auth 地址 (`http://localhost:3001`) 配合并携带返回的地址 `callbackUrl`。

- **开发与生产代理规范（Nginx / Vite）同步确认**:
  - 移除对未知 Nginx 网关地址错误假设导致部署中断的隐患代码。
  - 确认了现阶段本地发版 `npm run dev` 全量依赖 `vite.config.ts` 中的代理 (`/api/auth` 路由抛至 3001) 作为通讯基础；并在《AIASys 开发工作流指南》中留档记录本地与线上容器双轨部署结构的核心原理。

## 文档及规范构建 (Documentation & Standards)
