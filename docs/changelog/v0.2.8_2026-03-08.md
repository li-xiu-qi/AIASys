# Changelog v0.2.8 (2026-03-08)

## 新增功能

### GraphRAG 前端独立入口
- 新增受保护的 `/knowledge/graph`、`/knowledge/entities`、`/knowledge/communities` 路由
- 分析页侧栏新增“知识图谱”入口，保留原有“我的知识库”弹窗，不打断旧流程

### GraphRAG 知识图谱探索器
- 图谱概览页升级为基于 `G6` 的可视化探索器，支持总览图与问答子图切换
- 支持节点点击聚焦、搜索过滤、邻居高亮、社区着色以及 `force-atlas2 / concentric` 布局切换
- 保留文本构图入口和子图查询工作台，并将查询结果直接映射为可交互子图
- 页面壳体补齐滚动能力，上传构图区域不再因页面截断而难以访问
- 画布控制区改为左上角紧凑浮动工具栏，移除底部语义不清的白色条状控件
- 实体浏览页支持关键词搜索、实体类型筛选和单实体详情查看
- 社区分析页支持社区摘要浏览和社区报告生成

## 架构改进

### GraphRAG 前端数据层
- `apps/web/src/config/api.ts` 增加 `/api/graph/*` 端点定义
- 新增 `apps/web/src/lib/api/graphrag.ts` 类型化 API client
- 新增 `apps/web/src/types/graphrag.ts` 和共享页面壳体，统一 GraphRAG 展示逻辑

### GraphRAG 可视化协议
- 后端新增 `GET /api/graph/visualization` 端点，统一输出节点、边、社区和截断信息
- `POST /api/graph/query` 响应补齐 `subgraph` payload，前端可直接渲染问答命中的子图
- 小图社区检测增加降级逻辑，避免低节点数场景触发异常

### 文档提取模块化
- 后端新增共享 `document_extraction` 模块，统一 `knowledge` 与 `rag` 两条上传链路的文件提取能力
- 新增 `auto/basic/enhanced/docling` 四种提取模式，系统默认模式支持通过 `config.json` 配置
- `POST /api/knowledge/bases/{kb_id}/docs/upload` 和 `POST /api/rag/documents/upload` 均支持通过表单字段 `extraction_mode` 覆盖当前提取策略
