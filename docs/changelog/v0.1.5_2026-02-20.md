# 版本更新记录 v0.1.5

**更新日期**: 2026-02-20

## Claude

### 新增功能 (Features)

- **Skill 挂载路径修复**: 将容器内 skill 挂载路径从 `/root/.config/agents/skills` 修复为 `/root/.kimi/skills`
  - 原因：容器内运行的是 Kimi CLI，它从 `.kimi/skills` 路径原生扫描 skills
  - 影响文件：`ipython_tool.py`, `skills.py`, `manager.py`, `http_server.py`

- **Skill 渐进式加载支持**: 通过 `kimi-agent-sdk` 的 `skills_dir` 参数支持 skill 自动发现
  - `agent_service.py` 在创建 Session 时传入 `skills_dir`
  - Agent 现在能自动识别并使用挂载的 skills

### 优化与改进 (Improvements)

- **前端 Skill 管理界面优化**:
  - 删除右侧边栏 Skills 面板 (SkillsPanel.tsx)
  - 更新 SkillMarket 组件
  - 优化 useSkills hook，移除 enable/disable 逻辑（方案A：安装即启用）

### 文档完善 (Documentation)

- **更新 aiasys-workflow skill**: 添加 Kimi CLI Skill 加载机制说明
  - 说明 `.kimi/skills/` 和 `.agents/skills/` 的区别
  - 解释渐进式加载原理

### 测试 (Tests)

- **新增 Skill 相关测试**:
  - `test_skill_api.py`: Skill API 测试
  - `test_skill_docker_mount.py`: Docker 挂载测试
  - `test_skill_lifecycle.py`: Skill 生命周期测试

### 清理 (Chore)

- **移除 Supabase 相关文件**: 删除 infra/supabase/ 目录下的配置文件和数据文件
