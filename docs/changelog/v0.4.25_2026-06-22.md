# Changelog v0.4.25 - 2026-06-22

## li-xiu-qi

### 修复 (Bug Fixes)

#### 桌面版渲染进程崩溃后自动重启无效

- 现象：Windows + WSLg 环境下渲染进程崩溃后，自动以 `--disable-gpu` 重启通常无效，且弹窗阻断用户流程。
- 原因：自动重启逻辑在 Windows + WSLg 环境下无法解决根本兼容性问题，反而造成循环崩溃。
- 改动：移除自动重启与页面重载逻辑，崩溃后直接写入 `crash.log` 并退出，由用户自行排查。
- 文件变更
  - `apps/desktop/src/main.cjs`: 简化崩溃处理，崩溃即退出

#### 前端 preview server 代理 POST 请求卡死

- 现象：前端通过 preview server 代理向后端发送 POST 请求时，请求卡住无响应。
- 原因：预览服务器转发 POST 请求时未正确读取请求体，导致后端收不到完整数据。
- 改动：`local_preview_server.py` 的 `_proxy()` 方法先将请求体读入内存，再以 `data=body` 参数转发。
- 文件变更
  - `apps/web/scripts/committed/local_preview_server.py`: 修复 POST 代理 hang

#### 工作区创建流程状态缺失导致界面无响应

- 现象：新建工作区后，界面在部分阶段缺少明确状态提示，用户难以判断是否正在处理。
- 原因：运行时控制 hook 未在会话激活阶段设置 `activating_session` 状态；进度横幅在非数字进度阶段无法渲染状态文本。
- 改动：`useWorkspaceRuntimeControls.ts` 补充 `activating_session` 阶段与消息文本；`NewWorkspaceProgressBanner` 支持显示 message，并在无数字进度时渲染状态占位；类型定义补充 `binding_environment` / `waiting_runtime` 为 busy 阶段。
- 文件变更
  - `apps/web/src/pages/WorkspacePage/hooks/useWorkspaceRuntimeControls.ts`: 补充激活阶段状态
  - `apps/web/src/components/NewWorkspaceDialog/NewWorkspaceProgressBanner.tsx`: 支持 message 与状态占位渲染
  - `apps/web/src/types/workspace.ts`: 更新阶段标签
  - `apps/web/src/utils/newTaskLifecycleState.ts`: 补充 busy 阶段集合

#### 新建工作区对话框未校验提交条件

- 现象：未满足提交条件（如任务名称为空）时点击确认，仍会触发创建流程。
- 改动：`handleConfirm` 入口增加 `canSubmit` 前置检查，并输出调试日志辅助定位。
- 文件变更
  - `apps/web/src/components/NewWorkspaceDialog/index.tsx`: 增加提交前置校验与日志
