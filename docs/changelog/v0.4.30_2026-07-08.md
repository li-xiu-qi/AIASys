# Changelog v0.4.30 - 2026-07-08

## 修复 (Bug Fixes)

### 修复删除工作区时容器资源清理失败

#### 问题描述

- 现象：删除工作区时，`backend-spawn.log` 出现 `AttributeError: 'WorkspaceRegistryService' object has no attribute 'workspace_root'`，导致容器资源清理逻辑被异常跳过。删除接口虽返回 200，但清理步骤未实际执行，可能与删除后页面状态异常相关。
- 原因：`WorkspaceRegistryService._cleanup_workspace_containers` 中错误地访问了不存在的 `self.workspace_root` 属性。该类实际保存工作区根目录的属性为 `self.base_dir`。

#### 改动内容

- 将 `ContainerResourceService` 初始化参数从 `self.workspace_root` 修正为 `self.base_dir`，使删除工作区时能正确构造容器资源服务并执行容器清理。

#### 文件变更

- `apps/backend/app/services/workspace_registry.py`: 修正 `_cleanup_workspace_containers` 中 `ContainerResourceService` 的初始化参数。
