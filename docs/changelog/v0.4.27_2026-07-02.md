# Changelog v0.4.27 - 2026-07-02

## WWK-TJU

### 修复关闭 Thinking 后仍展示思考过程

#### 问题描述
- DeepSeek 等兼容接口在请求级关闭 Thinking 后，仍可能返回 `reasoning_content` 字段。
- 后端流式层原先只要收到 `reasoning_content` 就会下发 `content_type="think"`，导致前端在“关”状态下仍展示思考过程。
- SiliconFlow 的部分视觉 Thinking 模型不支持 `enable_thinking` 参数，按 Qwen 前缀泛化映射会导致 400 错误。
- OpenAI / Claude 的 thinking 参数存在模型级差异，按厂商泛化映射可能为 `gpt-5-pro`、旧 GPT-5、Claude Fable 5 / Sonnet 5 生成不支持的参数。

#### 改动内容
- 在 Agent runtime 流式层复用本轮 `LlmRequestOptions`，当 `thinking_disabled=True` 时抑制 reasoning 内容的 SSE 展示和历史落库。
- 增加 DeepSeek `deepseek-ai/DeepSeek-V4-Pro` 关闭 Thinking 的参数映射测试。
- 增加关闭 Thinking 时即使模型返回 reasoning delta 也不展示、不写入 `reasoning_content` 的回归测试。
- 将 SiliconFlow 作为独立 OpenAI-compatible provider 适配，仅对白名单内模型发送 `enable_thinking`。
- 增加 OpenAI 模型级 `reasoning_effort` 门控，避免 `gpt-5-pro` 和旧 GPT-5 收到不支持的 effort。
- 增加 Claude Fable 5 / Sonnet 5 adaptive thinking 识别，并在新 adaptive 模型开启 thinking 时移除不兼容采样参数。

#### 文件变更
- `apps/backend/app/services/agent/runtime_backends/aiasys/session_stream.py`: 根据请求级关闭状态过滤 reasoning 输出。
- `apps/backend/tests/test_aiasys_runtime_backend.py`: 增加关闭 Thinking 的 runtime 回归测试。
- `apps/backend/app/services/agent/runtime_backends/aiasys/llm_clients/thinking_mapper.py`: 增加 SiliconFlow thinking 参数白名单映射。
- `apps/backend/app/services/agent/runtime_backends/aiasys/llm_clients/anthropic_client.py`: 扩展 Claude adaptive thinking 模型识别。
- `apps/backend/tests/test_thinking_request_mapper.py`: 增加 DeepSeek、SiliconFlow、OpenAI、Claude 模型级 mapper 测试。

### 适配多模型厂商 Think / Reasoning 请求参数

#### 问题描述
- 不同模型厂商在 Chat Completions、Responses、Messages 协议下使用不同的 thinking / reasoning 参数形态。
- 原有实现把部分厂商参数散落在各 LLM client 内，难以覆盖 StepFun、Kimi、DeepSeek、Qwen、OpenAI、Claude 等差异。

#### 改动内容
- 新增统一的 thinking request mapper，将统一的 `LlmRequestOptions` 映射为厂商协议参数。
- 适配 OpenAI Responses、OpenAI-compatible Chat Completions、StepFun、Kimi、DeepSeek、Qwen / DashScope 和 Claude Messages 的 thinking / reasoning 参数。
- 支持请求级显式关闭 Thinking，并将关闭状态传到需要显式禁用的厂商参数中。
- 增加后端单元测试，覆盖各厂商映射和 Agent runtime 请求级关闭链路。

#### 文件变更
- `apps/backend/app/services/agent/runtime_backends/aiasys/llm_clients/thinking_mapper.py`: 新增多厂商 thinking / reasoning 参数映射。
- `apps/backend/app/services/agent/runtime_backends/aiasys/llm_clients/openai_client.py`: OpenAI-compatible Chat Completions 使用统一 mapper。
- `apps/backend/app/services/agent/runtime_backends/aiasys/llm_clients/codex_client.py`: OpenAI Responses 使用统一 mapper。
- `apps/backend/app/services/agent/runtime_backends/aiasys/llm_clients/anthropic_client.py`: Claude / Kimi Messages 兼容协议使用统一 mapper。
- `apps/backend/tests/test_thinking_request_mapper.py`: 新增厂商参数映射测试。
