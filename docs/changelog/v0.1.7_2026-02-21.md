# 版本更新记录 v0.1.7

**更新日期**: 2026-02-21

## 

### 新增功能 (Features)

- **OIDC 认证系统集成**:
  - 后端新增完整的 OIDC 认证流程支持，支持通过外部身份提供商登录
  - 新增 `AuthProvider` 类及 `OIDCAuthProvider` 实现，支持 OAuth2/OIDC 标准流程
  - 后端新增 `/api/auth/login` 和 `/api/auth/callback` 路由处理认证流程
  - 用户模型扩展支持 OIDC 用户信息映射（sub、email、name、roles）

- **前端认证模块重构**:
  - 重写 `AuthContext` 上下文，支持 OIDC 登录状态管理
  - 新增 `auth.ts` 配置文件集中管理认证相关配置
  - 新增 `oidc.ts` 库文件封装 OIDC 认证流程
  - 更新 `Login.tsx` 组件支持 OIDC 登录按钮
  - `authHeaders.ts` 优化 Token 管理和请求头注入

- **运维脚本工具集**:
  - 新增 `infra/scripts/` 目录包含 5 个运维脚本
  - `backup.sh`: 数据备份脚本，支持 storage 和 logs 目录打包
  - `deploy-local.sh`: 本地构建部署脚本
  - `health-check.sh`: 服务健康检查脚本
  - `logs.sh`: 日志查看脚本
  - `restart.sh`: 服务重启脚本

### 优化与改进 (Improvements)

- **Timeline UI 优化渲染**:
  - `StandardAgentRenderer` 和 `CodeAgentRenderer` 组件中隐藏了原生的 `tool_start` 和 `tool_output` 文本 Header 标签，避免与原有的折叠卡片（`CollapsibleToolBlock`）组件内容发生重叠和视觉冗余。去除了额外的 Label 渲染，提升了时间线界面的干净清晰度。

### 修复 (Bug Fixes)

- **修复 Sidebar 出现重复的 `TOOL START` 记录问题**:
  - 移除了 `useStreamEventHandler.ts` 中错误传递完整历史任务历史的数组行为（例如 `addStreamEvents(taskId, [...taskEventsMapRef.current[taskId]])`）。
  - 取而代之，改为每次仅增量传递当次触发解析到的单挑事件（`[toolCallEvent]` 和 `[toolResultEvent]` 等等），杜绝了随着工具执行次数增多而引发的 $O(N^2)$ 级别状态累加和 UI 复制渲染问题。

- **修复聊天输入框（InputArea）错误附加并展示「工作区旧文件」的问题**:
  - 移除了 `useAgentFileUpload.ts` 中错误的行为：将每次从后端拉取到的全量工作区文件 (`workspace files`) 同步覆盖到组件内部专门用于“发消息用暂存新附件区” (`state.files`) 的危险状态联动。真正实现了**“临时准备上传区”**与**“持久工作区文件展示 (`MultiTaskSidebar`)”**的主体职责解耦隔离。
  - 通过重构参数流转打通能力，在用户进行“切换不同历史会话”、“开启全新任务（`sessionOrchestrator`）”、“发送输入框内容后（`useExecutionSubmit`）”等生命周期节点时，强制下发调用 `clearFiles()` 以抹除上一次或上一条所残留的文件附件缓存状态。彻底解决历史生成的遗留图像/分析产物文件像“幽灵附件”一样挂载在新会话提示语下方的错误呈现。
- **修复 TypeScript 与 构建模块缺失错误**:
  - 修正了 `index.ts` 中因 Vite 构建环境 `import` 问题导致的 `getAuthHeaders` 路径报错。
  - 完成了 `useCodeExecutor` 分解后带来的零散 `any` 和类型不匹配（尤其是 `chatItems` 定义域）清理验证，使其在严苛的 `tsc` 构建下无报错。
- **修复全局“工作空间”弹窗无法正常读取旧文件的问题与命名重构**:
  - 修正了 `SessionReportsModal.tsx` 中读取挂载文件的 API Endpoint 错误调用的历史遗留问题（旧版误用了 `/api/files/.../workspace-files` 导致 404 返回空数组 `0 个文件`）。统一改用正规鉴权签名的 `API_ENDPOINTS.FILES_LIST` 及对应的下载预览接口，并映射数据格式结构使得跨旧会话图片与产物可正常加载浏览和执行独立删除。
  - 为避免概念引起困扰（现工作区面板和旧历史弹框概念混同），将原左侧边栏及弹窗顶部的表述由“工作空间 (Workspace)”更名为“资产总览 (Assets & Artifacts)”，概念区分更加明确干净。
- **useCodeExecutor Hooks 的有效隔离分解**:
  - 将巨大、行数超标近900行的 `useCodeExecutor/index.ts` 成功分拆重构为四个独立作用域功能模块： `useUIState`, `useSessionOrchestrator`, `useStreamEventHandler`, `useExecutionSubmit`。
  - 将原来的 `index.ts` 作为 Facade 输出模式统筹，大幅优化维护性与代码可读性。
- **优化工具详情悬浮窗显示格式 (`ToolPreviewPopover`)**:
  - **动态 Icon 与色彩映射**: 新增了针对工具名称前缀探测的功能（例如 `execute`, `search`, `file`, `task`）。工具不再仅仅使用单一普通的蓝色扳手图标，而是对应渲染如 `Terminal`（紫色）、`Globe`（青色）、`FileText`（翠绿） 和 `Network`（橘黄）等更符合操作语义的多彩图标块。
  - 修复了 JSON 参数解析展示冗余问题，把单一大块的 `JSON.stringify` 输出优化成了优雅遍历对象属性的形式。能够识别并支持字符串中代码或长段文本所含换行符的解析，使其渲染成更易读的单体语法区域，极大增强了像 `code` 这种参数属性的代码阅读体验。

- **技能目录结构重构**:
  - 将 `.github/skills/` 迁移到 `.agents/skills/aiasys-workflow/`，统一 AI 技能库存放位置
  - 整合 supabase-postgres-best-practices、uv-package-manager、vercel-composition-patterns、vercel-react-native-skills、web-design-guidelines 等技能
  - 新增技能主题文档目录 `topics/backend/`
  - 新增架构设计文档目录 `architecture/`

- **Docker 部署配置优化**:
  - 更新 `docker-compose.yml` 和 `docker-compose.prod.yml`，优化服务配置和环境变量传递
  - `Dockerfile.prod` 新增多阶段构建支持，优化镜像体积
  - 新增 `SECRET_KEY` 和 `JWT_SECRET` 环境变量配置支持
