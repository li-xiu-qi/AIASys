## Bug Fix

### 修复 Agent 首轮乱答与 GraphRAG 工具链误选模型
- 通过真实会话排查确认，用户消息和 session context 的传递本身正常，异常点出在 Agent 默认模型选择
- `agent_service` 之前会直接继承用户 LLM 配置中的 `default_model`，在当前配置下实际落到了 `deepseek-v3-aliyun`
- 对于需要稳定工具调用和 GraphRAG 检索的 Agent 场景，这个默认模型会导致首轮跑偏，出现自我介绍、列能力、扫目录等与用户任务无关的输出
- 现已改为：当请求未显式指定 `model_id` / `model` 时，Agent 优先自动选择 `kimi-for-coding` / `kimi-code` 这类更适合工具调用的模型；若不存在再回退到配置默认模型
- 同时收紧本地 Agent 提示词，并增加执行链路调试日志，便于继续观察首轮行为和工具调用轨迹

### 修复工作区目录型文件不显示
- 之前工作区文件列表接口只扫描 `workspace/` 目录下一层文件，像 `kb_documents/001_report.md` 这类目录内文件不会出现在工作区面板中
- 同时 `download`、`content`、`delete` 等接口会把文件名压成 basename，导致即使前端拿到带目录的路径，后续预览、编辑、下载、删除也会落到错误文件
- 现已将后端文件路由改为递归扫描 `workspace/`，并统一使用“工作区内相对路径”作为文件标识；相关接口已支持 `filename:path`
- 新增安全路径解析，明确拦截 `..`、绝对路径和越权路径，避免目录支持引入路径穿越风险
- 前端统一为工作区文件 URL 做按段编码，修复目录路径、中文名、空格文件名在下载和预览链路中的兼容性
