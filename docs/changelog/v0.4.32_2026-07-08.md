### 修复删除当前工作区后整个页面无法点击的问题

#### 问题描述

- 现象：在 v0.4.31 中，删除当前工作区后，Radix Dialog 的遮罩层（overlay）未被正确清理，导致 `document.body.style.pointerEvents` 仍停留在 `"none"`，整个应用页面无法点击，只能重启。
- 原因：删除工作区确认对话框在关闭瞬间即触发路由切换和工作区删除，Radix Dialog 的卸载动画与路由切换产生竞态，overlay 的清理逻辑被跳过。

#### 改动内容

- 在 `confirmDeleteWorkspace` 与 `confirmDeleteAllWorkspaces` 中：
  1. 先关闭对话框状态，预留 300ms 让 Radix Dialog 完成 overlay 卸载动画；
  2. 再执行删除 API 与路由 fallback；
  3. 在 `finally` 中兜底清除 `document.body.style.pointerEvents`，确保即使动画未完全结束也不会残留锁定。
- 删除完成后同步清理被删除工作区下所有会话的前端缓存状态。

#### 文件变更

- `apps/web/src/pages/WorkspacePage/components/WorkspaceLayout/hooks/useWorkspaceLifecycleActions.ts`

### 修复 bundled fnm 不识别裸 `lts` 版本别名的问题

#### 问题描述

- 现象：创建带 Node.js 环境的工作区时，部分场景下 `fnm install lts` 失败，提示 `The requested version is not installable: lts`。
- 原因：桌面版内置的 fnm 1.39.0 不识别裸 `lts` 别名，必须使用 `lts/*`。

#### 改动内容

- 在 `_normalize_node_version` 中将裸 `lts` 归一化为 `lts/*`。
- 将 `fnm install` 默认超时从 300s 延长至 600s，覆盖首次下载 Node.js 二进制的慢网络场景。

#### 文件变更

- `apps/backend/app/services/node_runtime.py`
