### 修复新建对话后 tokens/llm-selection 404、界面回退到旧会话的问题（issue #147）

#### 问题描述

- 现象：工作区内点击“新建对话”后，新会话界面短暂出现又回退到旧会话；新会话的 `tokens` 与 `llm-selection` 接口返回 404；连续点击会在后端遗留多个草稿会话。
- 原因：v0.4.34 的 release merge（`0448e6b`）在解决 `useSessionOrchestrator.ts` 冲突时取了功能分支上的旧版本，把主干上 4 个新建会话修复（`2a72168`、`847e1f2`、`5f8e4e6`、`78547fc`）静默回退；同一次合并还删除了后端回归测试 `test_execute_stream_binds_unbound_draft_to_workspace`。

#### 改动内容

- 恢复 `handleNewSession` 调用 `POST /api/sessions/create` 创建后端会话元数据（失败不阻塞 UI，前端继续乐观切换）。
- 恢复“新建对话总是创建全新 session”：不再复用 available-draft 预热草稿，空会话时也不复用当前 sessionId。
- 补回被合并删除的后端回归测试 `test_execute_stream_binds_unbound_draft_to_workspace`。

### 修复手动压缩被反增检测静默丢弃的问题

#### 问题描述

- 现象：用户手动触发上下文压缩（`/compact`）后无任何效果，`history.json` 不产生压缩快照与摘要。
- 原因：压缩的反增检测要求压缩后总 token 至少降低 5%，但检测口径包含 system prompt 等不参与压缩的固定开销；当固定开销占比高时（日常会话的常见情况），可压缩内容占比小，总降幅必然达不到 5%，导致 LLM 摘要已生成却被静默回滚。

#### 改动内容

- 手动强制压缩（`force=True`）跳过反增检测：用户显式触发且摘要已产出，不再因降幅不显著而丢弃。
- 自动触发（达到上下文阈值）保持原有反增检测不变，仍用于避免无效压缩循环。
- 补充单元测试覆盖两条路径：force 跳过检测、自动触发仍被拦截。

## 功能 (Features)

### 新增文件编辑保存快捷键（Ctrl+S / Cmd+S）

#### 问题描述

- 现象：在代码编辑器、Markdown/HTML/Code 预览的源编辑模式下修改文件后，只能通过点击“保存”按钮提交，缺少用户熟悉的全局保存快捷键。
- 影响：与 VS Code、Obsidian 等工具的交互习惯不一致，频繁点击保存按钮降低编辑效率。

#### 改动内容

- 新增通用 hook `useSaveShortcut`，统一拦截 `Ctrl+S`（Windows/Linux）和 `Cmd+S`（macOS），在满足条件时调用保存回调。
- `CodeEditorPanel` 接入快捷键：仅当文件可编辑、存在未保存修改且不在保存中时响应。
- `FilePreviewPanel` 的 Markdown / HTML / Code 源编辑模式接入快捷键：仅在源编辑模式、可编辑且有未保存修改时响应。
- 保存中状态自动忽略重复触发，避免重复提交。

### 新增 PowerShell 版本感知与提示词注入

#### 问题描述

- 现象：Windows 上 Agent 默认按 PowerShell 7（pwsh）语法生成命令，但用户系统实际是 PowerShell 5.1（powershell.exe），`&&`、三元运算符、`??` 等写法在 5.1 直接报错；经 WSL 互操作时语法混用报错更多。
- 影响：Agent 生成 shell 命令时无法感知宿主真实 PowerShell 版本，命令执行反复失败。

#### 改动内容

- 后端自动检测宿主 PowerShell 环境（pwsh 与 powershell 的路径、版本），生成 `${POWERSHELL_SECTION}` 提示词段落注入 system prompt：检测到 5.1 时明确禁用 `&&` / 三元 / `??` 等 7+ 专属语法，检测到 7+ 时提示经 WSL/旧版互操作需使用 5.1 兼容写法。
- 设置页「环境增强」面板新增 PowerShell 目标版本选择（自动 / 强制 5.1 / 强制 7+），偏好持久化到 `DATA_DIR/shell_preferences.json`，固定 5.1 时提示词中的路径与规则同步切换。
- 新增 `PUT /api/system/shell-environment/powershell-target` 接口保存目标版本。

## 架构重构 (Architecture)

### 前端弹窗底层从 Radix UI 迁移到 Base UI

#### 问题描述

- 现象：版本管理弹窗等场景下，点击详情后关闭弹窗，后续界面全部无法点击，页面处于卡死状态。
- 影响：Radix Dialog 的 overlay 清理与路由切换存在竞态，`document.body.style.pointerEvents` 等样式可能残留，导致全局交互失效。

#### 改动内容

- 将 `dialog`、`alert-dialog`、`sheet`、`dropdown-menu` 四个封装组件从 Radix UI 迁移到 Base UI。
- 保留原有业务代码 import 路径不变，通过封装层映射 Base UI 的 `DialogRoot` / `DialogPopup` / `DialogBackdrop` 等 primitive。
- 对 `DropdownMenuTrigger` 增加 `asChild` 兼容，对 `DialogContent` 增加 `onEscapeKeyDown` / `onPointerDownOutside` / `onOpenAutoFocus` 兼容，避免业务代码大面积修改。
- 前端构建通过，TypeScript 类型检查无错误。
