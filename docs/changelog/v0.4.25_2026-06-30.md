# Changelog v0.4.25 - 2026-06-30

## Clannad47

### 修复工具参数 JSON 容错解析

#### 问题描述

- 现象：模型输出工具调用参数时，如果 JSON 只存在可恢复格式问题，例如尾随逗号、单引号或缺少结尾括号，运行时会直接返回参数解析错误，导致工具调用失败。
- 原因：最终 tool arguments 解析边界只使用严格 `json.loads()`，没有针对模型常见的可恢复 JSON 片段做修复兜底。

#### 改动内容

- 新增 `json-repair` 后端依赖。
- 新增统一的工具参数解析 helper：严格解析失败后，再尝试使用 JSON repair 恢复。
- 将最终工具参数解析边界接入修复逻辑，同时保护不可恢复普通文本，避免被修复为空值后误执行。
- 新增回归测试覆盖尾随逗号、单引号、缺少结尾括号和完全非 JSON 文本。

#### 文件变更

- `apps/backend/app/services/agent/runtime_backends/aiasys/session_utils.py`: 新增工具参数 JSON 解析与修复 helper。
- `apps/backend/app/services/agent/runtime_backends/aiasys/session_tools.py`: 在最终工具参数解析边界接入统一 helper。
- `apps/backend/pyproject.toml` / `apps/backend/uv.lock`: 新增并锁定 `json-repair` 依赖。
- `apps/backend/tests/test_tool_argument_json_repair.py`: 新增工具参数 JSON 修复回归测试。
