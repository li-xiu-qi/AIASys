# Changelog v0.4.27 - 2026-07-01

## WWK-TJU

### 修复 Workspace PDF 预览无法显示

#### 问题描述
- 现象：Workspace 上传 PDF 后点击预览，界面可能显示 `{"detail":"Operation failed"}`；中文文件名 PDF 还可能触发后端 500。
- 原因：工作区文件预览 URL 使用了会话级下载接口；同时后端 inline 下载手写 `Content-Disposition`，非 ASCII 文件名会触发 HTTP header 编码错误。

#### 改动内容
- 修复中文文件名 PDF 预览 500。
- 工作区文件预览和下载在有 `workspaceId` 时优先使用 workspace-scoped download API。
- inline 文件响应改用 `FileResponse(filename=..., content_disposition_type="inline")` 生成合法的 `filename*=` 响应头。
- 扩展安全头中间件对 workspace/global workspace inline 预览路径的识别，避免 PDF iframe 被 frame blocking headers 拦截。
- 增加前端 URL 生成和后端中文文件名 / 安全头回归测试。

#### 文件变更
- `apps/web/src/utils/workspaceFiles.ts`: 支持 workspace-scoped 预览和下载 URL。
- `apps/backend/app/api/routes/workspaces_resources_files.py`: 修复 workspace/global workspace inline 下载响应头。
- `apps/backend/app/main.py`: 识别 workspace inline 文件预览路径。

### 修复 Agent 执行请求中的 Thinking 配置不生效

#### 问题描述
- 现象：Workspace 页面可以切换 Thinking 开关和 low / medium / high 强度，但发起 Agent 对话后，请求级配置没有进入最终 LLM runtime request options。
- 原因：前端提交链路没有把 `thinkingEnabled` / `thinkingEffort` 继续传给 `runAgentStream`，后端 `execute_stream()` 也没有把 `thinking_enabled` / `thinking_effort` 传给 `_get_config()`。

#### 改动内容
- 在 Workspace 提交流程中继续透传 Thinking 开关和强度配置。
- 在 Agent 执行链路中将请求级 Thinking 配置传入 `_get_config()`，使动态模型配置覆盖逻辑生效。
- 增加后端回归测试，覆盖 `execute_stream()` 参数透传和最终 `LlmRequestOptions` 生效。

#### 文件变更
- `apps/web/src/pages/WorkspacePage/hooks/useCodeExecutor/useExecutionSubmit.ts`: 透传 Thinking 配置到 `runAgentStream`。
- `apps/backend/app/services/agent/mixins/execution.py`: 透传请求级 Thinking 配置到 `_get_config()`。
- `apps/backend/tests/test_agent_thinking_request_options.py`: 新增请求级 Thinking 配置回归测试。
