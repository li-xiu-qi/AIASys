# Changelog v0.4.31 - 2026-07-08

## li-xiu-qi

### 功能 (Features)

### 文件变更面板升级为变更流时间线视图

#### 改动内容

- 后端文件历史服务新增变更事件聚合能力，将时间相近的文件历史条目按事件分组。
- 新增工作区级和全局工作区级变更事件查询 API：
  - `GET /api/workspaces/{workspace_id}/files/history/change-events`
  - `GET /api/workspaces/{workspace_id}/global-workspace/history/change-events`
- 前端"文件变更"侧边栏从文件列表改为变更流时间线，按事件展示最近的文件变更。
- 每个变更事件显示来源（用户/Agent/系统）、操作类型、涉及文件数量和时间。
- 保留原有能力：点击文件查看 diff、恢复到历史版本、查看完整历史。

#### 文件变更

- `apps/backend/app/services/file_history.py`: 新增 `ChangeEvent` 数据模型和 `list_change_events` 聚合方法。
- `apps/backend/app/api/routes/workspaces_resources_files.py`: 新增变更事件查询路由。
- `apps/backend/tests/test_change_events.py`: 新增变更事件聚合单元测试。
- `apps/web/src/components/layout/WorkspaceSidebar/FileChangesPanel.tsx`: 改造为变更流视图。
- `apps/web/src/components/layout/WorkspaceSidebar/ChangeEventCard.tsx`: 新增变更事件卡片组件。
- `apps/web/src/components/layout/WorkspaceSidebar/ChangeEventFileRow.tsx`: 新增事件内文件行组件。
- `apps/web/src/lib/api/fileChanges.ts`: 新增变更事件 API 和类型。
- `apps/web/src/config/api.ts`: 新增变更事件端点配置。
